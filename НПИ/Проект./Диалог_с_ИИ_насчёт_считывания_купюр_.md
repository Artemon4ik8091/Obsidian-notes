### Шаг 2: Сборка схемы
- Сделай простой "слот" из картона или 3D-печати (как в POS-терминале), чтобы банкнота проходила мимо сенсоров.
- Подключи компоненты к Arduino:

  - **TCS3200 цветовой сенсор**:
    - S0 -> Pin 8 (Arduino)
    - S1 -> Pin 9
    - S2 -> Pin 10
    - S3 -> Pin 11
    - OUT -> Pin 12
    - VCC -> 5V, GND -> GND

  - **IR-сенсор**:
    - OUT -> A0 (аналоговый пин)
    - VCC -> 5V, GND -> GND

  - **ML8511 UV-сенсор**:
    - OUT -> A1
    - EN -> 3.3V (или 5V, в зависимости от модуля)
    - VCC -> 3.3V, GND -> GND
    - UV-LED подключи к Pin 13 (через резистор 220Ω) для активации света.

  - **LCD (с I2C)**:
    - SDA -> A4
    - SCL -> A5
    - VCC -> 5V, GND -> GND

Используй breadboard для теста. Для UV: Размести UV-LED напротив UV-сенсора, чтобы банкнота проходила между ними.

### Шаг 3: Как работает
- IR-сенсор обнаруживает банкноту (когда она вставлена).
- UV-LED включается, UV-сенсор проверяет интенсивность отражённого UV-света (настоящие рубли имеют флуоресцентные метки, которые дают высокий сигнал; подделки — низкий).
- Цветовой сенсор измеряет RGB-значения для определения номинала (нужно калибровать под рубли: протестируй реальные банкноты и запиши диапазоны).
  - Примеры цветов рублей (примерные, калибруй сам):
    - 50 руб: Синий (высокий blue).
    - 100 руб: Оливковый (зелёный + жёлтый).
    - 500 руб: Фиолетовый (red + blue).
    - 1000 руб: Голубой.
    - 5000 руб: Красный (высокий red).
- Если UV OK и цвет совпадает — "Настоящая, номинал X руб". Иначе — "Подделка".
- Для магнитной проверки (опционально, +200 руб): Добавь Hall-сенсор (KY-003), но это усложнит — рубли имеют магнитные чернила в определённых местах.

### Шаг 4: Код на Arduino
Установи Arduino IDE, добавь библиотеки: LiquidCrystal_I2C (для LCD).

Вот базовый код (адаптирован из открытых туториалов). Он проверяет UV и цвет, выводит на LCD. Калибруй значения в if-условиях, протестировав банкноты (используй Serial.print для отладки RGB).

```cpp
#include <LiquidCrystal_I2C.h>

LiquidCrystal_I2C lcd(0x27, 16, 2);  // Адрес I2C может быть 0x3F

// Пины для TCS3200
#define S0 8
#define S1 9
#define S2 10
#define S3 11
#define sensorOut 12

// Пины для других
#define IR_PIN A0
#define UV_PIN A1
#define UV_LED 13

int redFreq = 0;
int greenFreq = 0;
int blueFreq = 0;

void setup() {
  lcd.init();
  lcd.backlight();
  lcd.print("Detector Ready");

  pinMode(S0, OUTPUT);
  pinMode(S1, OUTPUT);
  pinMode(S2, OUTPUT);
  pinMode(S3, OUTPUT);
  pinMode(sensorOut, INPUT);
  pinMode(IR_PIN, INPUT);
  pinMode(UV_PIN, INPUT);
  pinMode(UV_LED, OUTPUT);

  // Установка частоты 20% для TCS3200
  digitalWrite(S0, HIGH);
  digitalWrite(S1, LOW);

  Serial.begin(9600);
}

void loop() {
  if (digitalRead(IR_PIN) == LOW) {  // Банкнота обнаружена (LOW — в зависимости от сенсора)
    digitalWrite(UV_LED, HIGH);  // Включи UV-свет
    delay(500);  // Подожди

    // Читай UV
    int uvValue = analogRead(UV_PIN);
    bool isAuthentic = (uvValue > 500);  // Калибруй порог! >500 — настоящая (высокий сигнал от меток)

    // Читай цвета
    redFreq = getRed();
    greenFreq = getGreen();
    blueFreq = getBlue();

    Serial.print("R:"); Serial.print(redFreq);
    Serial.print(" G:"); Serial.print(greenFreq);
    Serial.print(" B:"); Serial.println(blueFreq);

    String nominal = "Unknown";
    if (redFreq > 20 && redFreq < 30 && blueFreq > 30 && blueFreq < 40) {  // Пример для 50 руб (синий) — калибруй!
      nominal = "50 RUB";
    } else if (redFreq > 40 && redFreq < 50 && greenFreq < 20) {  // Пример для 5000 руб (красный)
      nominal = "5000 RUB";
    } // Добавь для других номиналов

    lcd.clear();
    if (isAuthentic) {
      lcd.print("Authentic: " + nominal);
    } else {
      lcd.print("Fake!");
    }

    digitalWrite(UV_LED, LOW);
    delay(2000);  // Пауза
  }
}

// Функции для цветов
int getRed() {
  digitalWrite(S2, LOW);
  digitalWrite(S3, LOW);
  return pulseIn(sensorOut, LOW);
}

int getGreen() {
  digitalWrite(S2, HIGH);
  digitalWrite(S3, HIGH);
  return pulseIn(sensorOut, LOW);
}

int getBlue() {
  digitalWrite(S2, LOW);
  digitalWrite(S3, HIGH);
  return pulseIn(sensorOut, LOW);
}
```

# Ответ 2:

Конечно, отличная идея! Добавить режим калибровки — это логично, чтобы адаптировать детектор под реальные условия освещения и конкретные сенсоры. Мы сохраним калибровочные данные (средние RGB-значения для каждого номинала) в EEPROM Arduino (встроенная энергонезависимая память, чтобы данные не стирались при перезапуске). Для UV-проверки можно добавить пороговое значение, но давай сосредоточимся на цвете, как ты просил — UV оставим как есть (его порог можно откалибровать вручную в коде).

Я обновлю код: добавим кнопку для входа в режим калибровки (подключи кнопку к Pin 2, с pull-up резистором 10kΩ к 5V). При запуске или нажатии кнопки Arduino войдёт в калибровку: попросит вставить банкноты по порядку (50, 100, 500, 1000, 5000 руб), прочитает RGB несколько раз для усреднения и запишет в EEPROM. После калибровки — переключится в обычный режим.

Если калибровка уже сделана (проверим флаг в EEPROM), то сразу в обычный режим. Для простоты номиналы фиксированные: массив из 5 (можно расширить).

### Обновления в схеме
- Добавь кнопку: Один конец к Pin 2 (INPUT_PULLUP), другой к GND. Нажатие — LOW.
- EEPROM уже встроена в Arduino, ничего не нужно добавлять.

### Как калибровка работает
- Включи Arduino, нажми кнопку (или используй Serial-команду для теста).
- LCD покажет: "Калибровка: Вставь 50 руб".
- Вставь банкноту, IR-сенсор обнаружит, прочитает RGB (усреднит 5 измерений для точности).
- Сохранит в EEPROM: Для каждого номинала — red, green, blue (по 2 байта на значение, итого ~30 байт).
- Повтори для всех номиналов.
- После — флаг "калибровано" и перезапуск в обычный режим.
- В обычном: Читай RGB, сравнивай с сохранёнными (используй евклидово расстояние для ближайшего совпадения, чтобы учесть вариации).

### Обновлённый код
Установи библиотеку EEPROM (встроенная, но добавь #include <EEPROM.h>). Для LCD — как раньше.

```cpp
#include <LiquidCrystal_I2C.h>
#include <EEPROM.h>

LiquidCrystal_I2C lcd(0x27, 16, 2);  // Или 0x3F

// Пины для TCS3200
#define S0 8
#define S1 9
#define S2 10
#define S3 11
#define sensorOut 12

// Другие пины
#define IR_PIN A0
#define UV_PIN A1
#define UV_LED 13
#define CALIB_BUTTON 2  // Кнопка для калибровки

// Номиналы (в рублях)
const int nominals[] = {50, 100, 500, 1000, 5000};
const int numNominals = 5;

// Структура для хранения RGB в EEPROM (по 2 байта на цвет, но используем int)
struct ColorData {
  int red;
  int green;
  int blue;
};

ColorData calibData[numNominals];  // Массив калибровок

// Адреса в EEPROM
#define CALIB_FLAG_ADDR 0  // 1 байт: 1 если откалибровано
#define CALIB_DATA_ADDR 1  // С 1: данные (numNominals * sizeof(ColorData))

int redFreq = 0;
int greenFreq = 0;
int blueFreq = 0;

void setup() {
  lcd.init();
  lcd.backlight();
  Serial.begin(9600);

  pinMode(S0, OUTPUT);
  pinMode(S1, OUTPUT);
  pinMode(S2, OUTPUT);
  pinMode(S3, OUTPUT);
  pinMode(sensorOut, INPUT);
  pinMode(IR_PIN, INPUT);
  pinMode(UV_PIN, INPUT);
  pinMode(UV_LED, OUTPUT);
  pinMode(CALIB_BUTTON, INPUT_PULLUP);  // Кнопка с внутренним pull-up

  digitalWrite(S0, HIGH);
  digitalWrite(S1, LOW);  // 20% частота

  // Проверь флаг калибровки
  byte calibFlag = EEPROM.read(CALIB_FLAG_ADDR);
  if (calibFlag != 1 || digitalRead(CALIB_BUTTON) == LOW) {  // Если не откалибровано или кнопка нажата
    calibrationMode();
    EEPROM.write(CALIB_FLAG_ADDR, 1);  // Установи флаг
  } else {
    loadCalibration();  // Загрузи данные
  }

  lcd.clear();
  lcd.print("Detector Ready");
}

void loop() {
  if (analogRead(IR_PIN) < 500) {  // Банкнота обнаружена (порог для аналогового IR, откалибруй)
    digitalWrite(UV_LED, HIGH);
    delay(500);

    int uvValue = analogRead(UV_PIN);
    bool isAuthentic = (uvValue > 500);  // Порог UV — откалибруй вручную

    redFreq = getRed();
    greenFreq = getGreen();
    blueFreq = getBlue();

    Serial.print("R:"); Serial.print(redFreq);
    Serial.print(" G:"); Serial.print(greenFreq);
    Serial.print(" B:"); Serial.println(blueFreq);

    // Найди ближайший номинал по евклидову расстоянию
    int minDist = INT_MAX;
    int detectedNominal = -1;
    for (int i = 0; i < numNominals; i++) {
      int dr = redFreq - calibData[i].red;
      int dg = greenFreq - calibData[i].green;
      int db = blueFreq - calibData[i].blue;
      int dist = sqrt(dr*dr + dg*dg + db*db);  // Евклидово расстояние
      if (dist < minDist) {
        minDist = dist;
        detectedNominal = nominals[i];
      }
    }

    lcd.clear();
    if (isAuthentic && minDist < 50) {  // Порог расстояния — откалибруй (50 — пример)
      lcd.print("Authentic: ");
      lcd.setCursor(0, 1);
      lcd.print(detectedNominal);
      lcd.print(" RUB");
    } else {
      lcd.print("Fake!");
    }

    digitalWrite(UV_LED, LOW);
    delay(2000);
  }

  // Если кнопка нажата в loop, можно перезайти в калибровку
  if (digitalRead(CALIB_BUTTON) == LOW) {
    calibrationMode();
    EEPROM.write(CALIB_FLAG_ADDR, 1);
    delay(1000);  // Анти-дребезг
  }
}

// Режим калибровки
void calibrationMode() {
  lcd.clear();
  lcd.print("Calibration Mode");

  for (int i = 0; i < numNominals; i++) {
    lcd.clear();
    lcd.print("Insert ");
    lcd.print(nominals[i]);
    lcd.print(" RUB");

    while (analogRead(IR_PIN) >= 500) {  // Жди банкноту
      delay(100);
    }

    // Усредни 5 измерений
    int avgRed = 0, avgGreen = 0, avgBlue = 0;
    for (int j = 0; j < 5; j++) {
      avgRed += getRed();
      avgGreen += getGreen();
      avgBlue += getBlue();
      delay(200);
    }
    calibData[i].red = avgRed / 5;
    calibData[i].green = avgGreen / 5;
    calibData[i].blue = avgBlue / 5;

    // Сохрани в EEPROM
    int addr = CALIB_DATA_ADDR + i * sizeof(ColorData);
    EEPROM.put(addr, calibData[i]);

    lcd.clear();
    lcd.print("Calib ");
    lcd.print(nominals[i]);
    lcd.print(" Done");
    delay(2000);

    while (analogRead(IR_PIN) < 500) {  // Жди удаления банкноты
      delay(100);
    }
  }

  lcd.clear();
  lcd.print("Calib Complete!");
  delay(2000);
}

// Загрузи калибровку из EEPROM
void loadCalibration() {
  for (int i = 0; i < numNominals; i++) {
    int addr = CALIB_DATA_ADDR + i * sizeof(ColorData);
    EEPROM.get(addr, calibData[i]);
  }
}

// Функции для цветов (как раньше)
int getRed() {
  digitalWrite(S2, LOW);
  digitalWrite(S3, LOW);
  return pulseIn(sensorOut, LOW);
}

int getGreen() {
  digitalWrite(S2, HIGH);
  digitalWrite(S3, HIGH);
  return pulseIn(sensorOut, LOW);
}

int getBlue() {
  digitalWrite(S2, LOW);
  digitalWrite(S3, HIGH);
  return pulseIn(sensorOut, LOW);
}
```

### Пояснения
- **Калибровка**: Автоматически усредняет измерения для точности. Используй Serial Monitor, чтобы видеть значения во время теста.
- **Сравнение**: Евклидово расстояние — простой способ найти "ближайший" цвет. Порог 50 — пример; протестируй и подкорректируй (если расстояние больше — "Unknown").
- **UV**: Оставил как есть, но если хочешь калибровать, добавь аналогично (средний uvValue для настоящих).
- **Тесты**: Сначала откалибруй с реальными банкнотами (в одном освещении). Если данные в EEPROM "грязные", сбрось флаг: В setup добавь EEPROM.write(CALIB_FLAG_ADDR, 0); один раз.
- **Улучшения**: Если нужно больше номиналов или UV-калибровка — скажи. Для IR я перешёл на analogRead (предполагая аналоговый сенсор; если цифровой — верни digitalRead).
