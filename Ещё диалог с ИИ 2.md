### Основные компоненты для сборки:
- **Arduino Nano** (как мозг проекта).
- **Цветовой сенсор TCS230 или TCS3200** — для определения цвета купюры (разные номиналы имеют разные преобладающие цвета, например, 100 рублей — синий, 500 — зелёный).
- **IR-сенсор (инфракрасный)** — для детекции присутствия купюры (определяет, когда купюра проходит через слот).
- **Опционально: УФ-LED и фотодиод** — для проверки флуоресценции (некоторые купюры светятся под УФ).
- **Для размера: Линейный CCD-сенсор (типа TSL1401) или несколько IR-сенсоров в ряд** — чтобы измерить ширину/длину по времени прохождения.
- **Дисплей (например, 16x2 LCD)** — для вывода номинала.
- **Дополнительно: Белые LED для подсветки, картон или 3D-печатный корпус** для слота, куда вставляется купюра.
- **Питание и провода** (breadboard для прототипа).

### Принцип работы:
1. **Детекция присутствия**: IR-сенсор фиксирует, когда купюра входит в слот (луч прерывается). Это запускает сканирование.
2. **Определение цвета**: TCS230 измеряет интенсивность красного, зелёного и синего (RGB) света, отражённого от купюры. В коде задаются диапазоны значений для каждого номинала (нужно калибровать вручную для российских рублей, тестируя реальные купюры). Например:
   - Для 100 рублей: Red 25-30, Green 20-25, Blue 35-40 (примерные значения; подстраивайте под ваши измерения).
   - Сенсор выдаёт частоту сигнала пропорционально интенсивности — это читается через pulseIn() в коде.
3. **Измерение размера**: Если используете CCD, он сканирует линию пикселей и определяет ширину тени от купюры. Альтернатива — моторчик для протяжки купюры и тайминг прохождения мимо IR.
4. **УФ-проверка (опционально)**: Включаете УФ-LED, фотодиод измеряет отражённый свет — для отличия подделок.
5. Когда купюра выходит (IR возвращается в норму), Arduino выводит номинал на экран и, optionally, подсчитывает сумму.

### Пример кода (базовый скетч на Arduino IDE):
```cpp
#include <LiquidCrystal.h>

LiquidCrystal lcd(4, 6, 7, 8, 9, 10);  // Пины для LCD

#define S0 2
#define S1 3
#define S2 11
#define S3 12
#define sensorOut 13
#define IR A0

int redFreq = 0, greenFreq = 0, blueFreq = 0;
int total = 0;  // Сумма

void setup() {
  pinMode(S0, OUTPUT);
  pinMode(S1, OUTPUT);
  pinMode(S2, OUTPUT);
  pinMode(S3, OUTPUT);
  pinMode(sensorOut, INPUT);
  pinMode(IR, INPUT);
  
  digitalWrite(S0, HIGH);  // Масштаб частоты 20%
  digitalWrite(S1, LOW);
  
  lcd.begin(16, 2);
  lcd.print("Вставьте купюру");
}

void loop() {
  if (digitalRead(IR) == HIGH) {  // Купюра обнаружена
    redFreq = getRed();
    greenFreq = getGreen();
    blueFreq = getBlue();
    
    if (redFreq > 20 && redFreq < 25 && blueFreq > 30 && blueFreq < 35 && greenFreq > 30 && greenFreq < 35) {
      lcd.clear();
      lcd.print("100 рублей!");
      total += 100;
    }  // Добавьте условия для других номиналов
    
    delay(500);  // Анти-дребезг
    while (digitalRead(IR) == HIGH);  // Ждём выхода купюры
  }
  lcd.setCursor(0, 1);
  lcd.print("Сумма: " + String(total));
}

int getRed() {
  digitalWrite(S2, LOW);
  digitalWrite(S3, LOW);
  return pulseIn(sensorOut, LOW);
}
// Аналогично для getGreen() и getBlue()
```
Это адаптация из проектов; подкорректируйте пины под Nano и добавьте больше номиналов. Калибровка — ключ: протестируйте реальные купюры в одинаковых условиях освещения.

### Ограничения:
- Точность ниже банкоматной: зависит от калибровки, освещения и износа купюр. Не распознаёт подделки полноценно.
- Arduino Nano может не потянуть сложную обработку (например, без камеры для OCR), но для цвета и IR хватит.
- Для улучшения: Добавьте камеру (типа OV7670) с библиотекой OpenCV (но это потребует более мощного микроконтроллера, как ESP32).
- Стоимость: Около 500-2000 рублей на компоненты.

Если нужно, могу уточнить схему или код под конкретные номиналы.


## Ответ 2
Да, реализовать режим калибровки на Arduino Nano вполне возможно и даже рекомендуется, чтобы повысить точность распознавания. Поскольку точные значения RGB (и других параметров) зависят от освещения, качества сенсора TCS230/TCS3200 и состояния купюр, калибровка позволит вам эмпирически определить диапазоны для каждого номинала (например, 100, 500, 1000 рублей и т.д.). 

Вы можете сделать это двумя способами:
1. **Отдельный скетч для калибровки**: Простой и безопасный вариант. Загружаете его на Arduino, вставляете купюры по очереди, смотрите значения в Serial Monitor (в Arduino IDE: Tools > Serial Monitor, baud rate 9600). Записываете средние/диапазоны значений вручную, затем вписываете их в условия основного скетча (как в примере из предыдущего ответа).
2. **Интегрированный режим в основном скетче**: Добавляете переключатель (кнопку на пине) или команду через Serial. Когда режим активен, при вставке купюры выводятся параметры вместо определения номинала. Это удобнее, но код станет чуть сложнее.

Я рекомендую начать с отдельного скетча — он проще для тестирования. Вот как это работает: 
- При обнаружении купюры (через IR-сенсор) скетч читает RGB-частоты с TCS230.
- Выводит их в Serial Monitor в формате: "Red: X, Green: Y, Blue: Z".
- Опционально: Добавьте измерение размера (если используете CCD или несколько IR) или УФ.
- Процедура: Вставьте купюру одного номинала 5-10 раз (в разных ориентациях, если нужно), запишите мин/макс/средние значения. Для каждого номинала создайте диапазоны с запасом (например, Red: 20-30).
- Ограничения: Делайте калибровку в тех же условиях освещения, что и использование. Если купюры изношенные, учтите разброс.

### Схема подключения (та же, что в основном скетче):
- TCS230: S0 к D2, S1 к D3, S2 к D11, S3 к D12, OUT к D13.
- IR-сенсор: К A0 (аналоговый, но читаем как digital для простоты; если analog, используйте threshold).
- LCD опционально, но для калибровки хватит Serial.
- Не забудьте подключить белые LED для стабильной подсветки.

### Отдельный скетч для калибровки (загрузите в Arduino IDE):
```cpp
#define S0 2
#define S1 3
#define S2 11
#define S3 12
#define sensorOut 13
#define IR A0  // IR-сенсор для детекции купюры

int redFreq = 0, greenFreq = 0, blueFreq = 0;

void setup() {
  pinMode(S0, OUTPUT);
  pinMode(S1, OUTPUT);
  pinMode(S2, OUTPUT);
  pinMode(S3, OUTPUT);
  pinMode(sensorOut, INPUT);
  pinMode(IR, INPUT);
  
  // Установка масштаба частоты TCS230 (20% для баланса чувствительности)
  digitalWrite(S0, HIGH);
  digitalWrite(S1, LOW);
  
  Serial.begin(9600);
  Serial.println("Калибровка: Вставьте купюру для измерения.");
}

void loop() {
  // Проверяем наличие купюры (предполагаем, что IR LOW при прерывании луча)
  if (digitalRead(IR) == LOW) {  // Или analogRead(IR) < threshold, если analog
    delay(200);  // Дебounce, чтобы купюра стабильно вошла
    
    redFreq = getRed();
    greenFreq = getGreen();
    blueFreq = getBlue();
    
    // Вывод в Serial
    Serial.print("Red: ");
    Serial.print(redFreq);
    Serial.print(", Green: ");
    Serial.print(greenFreq);
    Serial.print(", Blue: ");
    Serial.println(blueFreq);
    
    delay(1000);  // Пауза для чтения, затем ждём выхода купюры
    while (digitalRead(IR) == LOW);  // Ждём, пока купюра не выйдет
    Serial.println("Вставьте следующую купюру.");
  }
}

int getRed() {
  digitalWrite(S2, LOW);
  digitalWrite(S3, LOW);
  return pulseIn(sensorOut, LOW);
}

int getGreen() {
  digitalWrite(S2, HIGH);
  digitalWrite(S3, HIGH);
  return pulseIn(sensorOut, LOW);
}

int getBlue() {
  digitalWrite(S2, LOW);
  digitalWrite(S3, HIGH);
  return pulseIn(sensorOut, LOW);
}
```
- **Как использовать**: После загрузки откройте Serial Monitor. Вставьте купюру — увидите значения. Для каждого номинала соберите данные и рассчитайте диапазоны (например, в Excel: мин-5% до макс+5% для запаса).
- **Расширение**: Если добавляете УФ (фотодиод на A1 с УФ-LED на D5), в loop добавьте: `digitalWrite(5, HIGH); int uv = analogRead(A1); digitalWrite(5, LOW); Serial.print(", UV: "); Serial.println(uv);`.
- **Для размера**: Если используете TSL1401 (CCD), добавьте код для чтения ширины — но это усложнит скетч; начните с цвета.

### Интеграция в основной скетч:
Если хотите режим калибровки в основном коде, добавьте кнопку (на D4, с pull-up). В setup: `pinMode(4, INPUT_PULLUP);`. В loop: 
```cpp
bool calibMode = (digitalRead(4) == LOW);  // Если кнопка нажата — калибровка
if (digitalRead(IR) == HIGH) {  // Купюра
  redFreq = getRed();  // etc.
  if (calibMode) {
    // Вывод в Serial как выше
  } else {
    // Логика определения номинала
  }
}
```
Это позволит переключаться без перезагрузки.

Если значения сильно разнятся или сенсор нестабильный, попробуйте усреднять несколько чтений (в loop: читайте 3 раза и берите среднее). Если нужны уточнения по коду или добавление параметров (размер/УФ), дайте знать!
